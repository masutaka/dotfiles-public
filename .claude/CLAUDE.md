# CLAUDE.md

## 必須事項

- 日本語で回答すること
- ユーザーからの指示や仕様に疑問を感じたら、作業を中断し、何でも質問すること
- Markdown の強調表示や絵文字は、必要最低限に留めること

## コーディング原則

### 開発プロセス

- TDD は t-wada が推奨する進め方に従うこと
    - Red-Green-Refactor のサイクルを厳守し、小さなステップで進める
- リファクタリングは Martin Fowler が推奨する進め方に従うこと
    - 動作を変えずに構造を改善し、機能追加と分離する

### 設計原則

- YAGNI: 今必要じゃない機能は作らない
- DRY: 同じコードを繰り返さない
- KISS: シンプルに保つ
- 最小限の変更: 既存構造の活用を優先し、新しい構造を作らない
- セキュリティは OWASP のガイドラインに従うこと

### オブジェクト指向設計

- SOLID 原則を遵守すること
- Tell, Don't Ask: オブジェクトに状態を問い合わせて判断するのではなく、振る舞いを依頼すること

## 操作規則

### ファイルとディレクトリ

- シェルのエイリアスをバイパスするために、cp, mv, rm の代わりに \cp, \mv, \rm を使用すること
- テキストファイル（非バイナリファイル）の終端は空行にすること
- テキストファイルの行末に存在する、不要な空白やタブ文字は除去すること

### GitHub

- gh CLI を使用すること

### パッケージ

- npm パッケージは "npm install" または "npm install --save-dev" でインストールすること
